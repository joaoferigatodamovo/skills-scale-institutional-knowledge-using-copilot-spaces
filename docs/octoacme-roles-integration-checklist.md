
# OctoAcme — Roles & Personas Integration Checklist

Use this checklist when onboarding a new team member or integrating a new role into an existing project.

## Pre-Project Setup

- [ ] All required personas identified for the project scope
- [ ] RACI matrix created (Responsible, Accountable, Consulted, Informed)
- [ ] Communication plan drafted with frequency and participants
- [ ] Project board configured with role-specific swimlanes or labels (if applicable)
- [ ] Shared documentation location established (e.g., project repo, wiki)
- [ ] Success metrics baseline captured by Data Analyst

## Kickoff & Orientation

- [ ] Project charter reviewed with all personas present
- [ ] Each role's responsibilities clearly articulated
- [ ] Decision-making authority established for each persona
- [ ] Escalation paths reviewed and documented
- [ ] Calendar holds set for recurring ceremonies (standups, syncs, retros)
- [ ] Access to shared tools granted (project board, docs, dashboards)

## Sprint 1 Readiness

- [ ] Backlog refined by Product Manager with input from key personas
- [ ] User research / customer context provided by UX Designer and Support Lead
- [ ] Acceptance criteria written with testing perspective (QA alignment)
- [ ] Metrics dashboard created by Data Analyst for monitoring progress
- [ ] Risk register initiated by Project Manager with cross-functional input
- [ ] Definition of Done agreed upon with all personas

## Ongoing Integration

**Weekly:**
- [ ] Standups include participation from core personas
- [ ] Blockers identified by Scrum Master and escalated appropriately
- [ ] Status communicated to all stakeholders

**After Each Sprint:**
- [ ] Demo includes contributions from all personas (design, metrics, support perspective)
- [ ] Retrospective solicits feedback from all functions
- [ ] Action items assigned to responsible personas with clear owners

**Monthly / Quarterly:**
- [ ] Cross-functional sync to review emerging gaps or inefficiencies
- [ ] Success metrics reviewed by Data Analyst with Product Manager
- [ ] Documentation updated to reflect any process changes

## Feedback & Continuous Improvement

- [ ] Gather feedback from each persona on collaboration effectiveness
- [ ] Document any process gaps or inefficiencies
- [ ] Update RACI or communication plan as needed
- [ ] Share learnings in project retrospective and org-wide retros
- [ ] Iterate on ceremony structure based on team feedback

## Handoff & Close-Out

- [ ] All personas contribute to project retrospective
- [ ] Data Analyst produces final impact report
- [ ] Post-mortem (if applicable) includes Support Lead feedback on customer impact
- [ ] Lessons learned documented and shared with organization
- [ ] Knowledge transfer completed for sustained operations (if ongoing support needed)
- [ ] Update octoacme-roles-and-personas.md with any refinements to role definitions
